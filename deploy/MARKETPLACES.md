# API Marketplaces — Overview & Setup Guide

Your API suite can be listed on multiple marketplaces simultaneously.
The gateway (`port 8000`) handles auth for all of them automatically.

## Available Marketplaces

| Marketplace | Audience | Revenue Share | Auth Method | Difficulty |
|---|---|---|---|---|
| **RapidAPI** | Largest, 4M+ devs, global | 20% | X-RapidAPI-Key + Proxy-Secret | Medium |
| **APILayer** | Business/enterprise focus | 20-30% | apikey header | Easy |
| **Zyla API Hub** | Developer community | 20% | X-Zyla-Api-Key header | Easy |
| **API.market** | Growing marketplace | 15% | X-Api-Key header | Easy |
| **Postman API Network** | 20M+ developers | Free listing | Postman collection | Easy |
| **APIs.guru** | Open source discovery | Free | No auth | Easy |
| **Eden AI** | AI/data-focused | ~25% | API key | Medium |

## Recommendation

Start with these 3 for maximum reach:
1. **RapidAPI** — most traffic, most customers
2. **APILayer** — business customers, higher ARPU
3. **Postman API Network** — free, great for discovery

## 1. RapidAPI Setup

**URL**: https://rapidapi.com/provider

### Steps:
1. Create provider account at rapidapi.com/provider
2. Create New API → name: "API Suite by [Your Brand]"
3. Go to **Settings → Security** → copy the **Proxy Secret** 
4. Paste it as `RAPIDAPI_PROXY_SECRET` in your `.env`
5. Set Base URL: `https://your-domain.com` (nginx handles it)
6. Add endpoints (see `RAPIDAPI_LISTING.md` for full listing)
7. Set pricing plans (see below)

### How it works:
RapidAPI injects two headers on every request:
- `X-RapidAPI-Key` — the user's personal key
- `X-RapidAPI-Proxy-Secret` — your secret (proves it came from RapidAPI)
- `X-RapidAPI-Subscription` — the plan name ("BASIC", "PRO", etc.)

Your gateway validates these automatically.

### Pricing Plans (RapidAPI):
```
Free:   0$/month  → 100 requests/day
Basic:  9$/month  → 10,000 req/month  (~100 req/min burst)
Pro:    29$/month → 100,000 req/month (~300 req/min burst)
Ultra:  99$/month → unlimited          (1000 req/min burst)
```

## 2. APILayer Setup

**URL**: https://apilayer.com/marketplace

### Steps:
1. Create account at apilayer.com
2. Submit your API at apilayer.com/marketplace/submit
3. APILayer team reviews (1-3 business days)
4. Users authenticate with `apikey: YOUR_KEY` header
5. Add keys to your `.env` `API_KEYS` list

### How it works:
APILayer injects the `apikey` header. Your gateway reads it and matches against `API_KEYS`.

---

## 3. Zyla API Hub Setup

**URL**: https://zylalabs.com/api-marketplace/create-api

### Steps:
1. Sign up as provider
2. Submit your API with base URL and endpoints
3. Zyla injects `X-Zyla-Api-Key` header
4. Add your keys to `.env` `API_KEYS` list

---

## 4. API.market Setup

**URL**: https://api.market/create

### Steps:
1. Register as provider
2. Add your API with base URL
3. Uses `X-Api-Key` header (already supported by gateway)

## 5. Postman API Network (Free)

**URL**: https://www.postman.com/explore

### Steps:
1. Create a Postman workspace
2. Import the collection: `deploy/postman-collection.json` (generated by `generate-postman.sh`)
3. Publish to Postman Public API Network
4. No auth required for the listing — users still need your API key to call

**To generate the Postman collection:**
```bash
cd /srv/apis/deploy
./generate-postman.sh > postman-collection.json
# Then import to Postman
```

## Multi-Marketplace Configuration

All marketplaces work simultaneously because the gateway checks headers in order:

1. `X-Admin-Secret` → admin
2. `X-RapidAPI-Proxy-Secret` + `X-RapidAPI-Key` → RapidAPI
3. `apikey` → APILayer / direct  
4. `X-Api-Key` / `X-API-Key` → Zyla / API.market / Postman
5. `Authorization: Bearer KEY` → OAuth-style
6. No key → anonymous (if `ALLOW_ANONYMOUS=true`)

You don't need to change any code — just fill in the right `.env` values for each marketplace.
